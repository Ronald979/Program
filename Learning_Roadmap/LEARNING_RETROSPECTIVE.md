# Learning Retrospective

Last updated: 2026-06-03

## Retrospective Rule

每個階段結束後，都要補一段復盤。

復盤不是寫感想，而是回答：

1. 我完成了什麼？
2. 我原本以為懂，但其實沒懂的是什麼？
3. 哪些 bug 或卡點暴露了前置知識不足？
4. 哪些內容可以寫進 CV？
5. 哪些內容可以做成 PPT？
6. 下一階段要避免什麼？

## Phase 0 Retrospective - Workspace and Resume System

Date: 2026-06-03

Completed:

- 將工作區分成 `Learning_Material`、`CV`、`Learning_Roadmap`。
- 將 ERP / Simray 定位為過去工作經歷。
- 將 personal projects 定位為履歷簡短摘要與獨立 PPT 展開。
- 建立 master resume、personal project index、PPT notes。
- 建立 AI Agent monitor，讓每次任務開始與結束都能同步工作區狀態。

What is clearer now:

- ERP 是工作經歷，不應該放到 personal project 裡。
- 個人 project 在 resume 只需要名稱、目的與關鍵技術。
- 詳細架構、流程、測試、debug、成果應該放在 PPT 或 project case study。
- `Learning_Roadmap` 是未來每次開始學習前的入口。
- `AI_Agent_Monitor` 是未來 AI Agent 同步狀態的入口，避免檔案移動或 project 進度更新後失去上下文。

- Personal projects 還大多是 planned/draft 狀態。
- 尚未產生實際 PPT 檔。
