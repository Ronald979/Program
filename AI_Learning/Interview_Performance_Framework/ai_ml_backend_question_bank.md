# AI / ML / 後端 面試題庫（對應你目前方向）

本題庫設計為「可直接背、可直接練、可直接講」三軸：  
- 先背核心觀念  
- 再做題目演練  
- 最後能用「問題→做法→成果」三句完成口語化回答

> 你可依「每日 2 小時版 / 4 小時版」把題庫拆進每日時段。

## A. AI 類（含 LLM）
1. 什麼是 `RAG`？為什麼比直接讓 LLM 生回答更穩定？  
   - 重點：檢索前處理、召回、重排序、減少幻覺、可觀測性。

2. 你怎麼做 LLM 的本地部署？部署流程三步寫給我。  
   - 重點：環境、模型載入、推論 API、資源限流、錯誤重試。

3. 什麼是 `quantization`？AWQ/GGUF/4-bit 有什麼差別？  
   - 重點：記憶體占用、推論吞吐、精度損失、硬體適配。

4. 什麼是 LoRA/PEFT？和 full fine-tuning 差在哪？  
   - 重點：可訓練參數比例、速度、成本、風險。

5. RAG 的 `chunk size` 和 `overlap` 怎麼選？  
   - 重點：檢索粒度、上下文完整度、重複、索引體積。

6. 你怎麼設計 Prompt？有什麼常見 anti-pattern？  
   - 重點：system/user 區隔、禁止幻覺、輸出 schema、few-shot。

7. LLM 服務的冷啟動與高延遲問題怎麼處理？  
   - 重點：warm-up、batching、限流、快取、降級策略。

8. 怎麼做 LLM 專案的 eval？你會看哪些指標？  
   - 重點：人工評估、主觀指標、關鍵詞精準率、任務成功率。

9. 你有沒有遇過資料洩漏（data leakage）？如何發現？  
   - 重點：split 方式、hash 去重、時間切分、泄漏檢查。

10. GPU 只有 4GB 時，如何決定可行模型與 batch 設定？  
   - 重點：模型大小估算、KV cache、長度限制、並發與併發控制。

## B. ML 基礎與其他模型
1. Embedding/向量檢索為何比關鍵字檢索更好？  
   - 重點：語義相似、向量空間、近似最近鄰、cosine 與內積。

2. 什麼是 cosine similarity？為什麼常用於向量檢索？  
   - 重點：方向比對 vs 幅度差異、正規化。

3. 訓練資料不平衡你會怎麼處理？  
   - 重點：重採樣、class weight、閾值調整、PR-AUC。

4. 分類問題的 Precision / Recall / F1 各在乎什麼？  
   - 重點：誤判成本、商業權衡。

5. 簡述你對 CV 預處理（augmentation / resize / normalization）的認知。  
   - 重點：泛化、過擬合、輸入一致性。

6. 語音模型常見前處理步驟？  
   - 重點：降噪、重採樣、分幀、MFCC。

7. 什麼情況你會選用 Logistic / XGBoost / DNN？  
   - 重點：可解釋性、特徵數量、資料量、上線維運。

8. 什麼是過擬合與欠擬合？有何偵測方式？  
   - 重點：訓練/驗證曲線、正則化、早停。

9. 什麼是 ROC-AUC 與 PR-AUC，何時使用各自較合適？  
   - 重點：class imbalance、排序解釋。

10. 你怎麼做 ML 服務漂移（data drift）監控？  
   - 重點：輸入分布、特徵監控、警示門檻。

## C. 後端/系統題（AI 專職常考的工程題）
1. 你會怎麼設計一個「多模型推論 API Gateway」？  
   - 重點：路由、佇列、超時、fallback、降載、metrics。

2. API 常見的可用性指標有哪幾個？  
   - 重點：可用率、P95/P99、error rate、吞吐量、資源飽和度。

3. 什麼是佇列（queue）在系統中的角色？  
   - 重點：解耦、削峰、異步處理、死信隊列。

4. 如何設計簡單但不崩潰的日誌與監控？  
   - 重點：trace id、關鍵字段、結構化 logging、告警。

5. 什麼是快取擊穿 / 打穿 / 雪崩？  
   - 重點：TTL、隨機過期、熔斷、限流。

6. 你會怎麼處理併發請求下的共用模型資源？  
   - 重點：模型實例池、排隊、批次、請求降級。

7. 什麼是 ACID？與 BASE 有何差異？  
   - 重點：一致性、可用性與分散式取捨。

8. 什麼是 rate limiting？要怎麼設計？  
   - 重點：固定窗/滑動窗、ip/uid token bucket。

9. 你會如何設計一個最小可用的 Prompt 平台 API？  
   - 重點：認證、版本管理、審計、版本回滾、A/B。

10. 一則「面試官追問」：你剛剛的設計若有一半節點故障怎麼辦？  
    - 重點：重試、超時、冗餘、降級訊息、可觀測性。

## 每日練習建議
- 2 小時版：每天至少 1 題 AI 或 ML 題 + 1 題後端/系統題  
- 4 小時版：每天至少 2 題 AI/ML 題 + 1 題後端題 + 1 次口頭白板  
- 每週抽 1 天做「整題庫口說日」，只做答不做新題，檢查敘事邏輯
