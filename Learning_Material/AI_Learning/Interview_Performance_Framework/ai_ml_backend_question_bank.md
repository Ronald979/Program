# AI / ML / 後端 / 韌體 / FPGA 面試題庫（對應你目前方向）

本題庫設計為「可直接背、可直接練、可直接講」三軸：  
- 先背核心觀念  
- 再做題目演練  
- 最後能用「問題→做法→成果」三句完成口語化回答

> 你可依「每日 2 小時版 / 4 小時版」把題庫拆進每日時段。

## A. AI 類（含 LLM）
1. 什麼是 `RAG`？為什麼比直接讓 LLM 生回答更穩定？  
   - 重點：檢索前處理、召回、重排序、減少幻覺、可追溯性。

2. 你怎麼做 LLM 的本地部署？部署流程三步寫給我。  
   - 重點：環境、模型載入、推論 API、資源限流、錯誤重試。

3. 什麼是 `quantization`？AWQ/GGUF/4-bit 有什麼差別？  
   - 重點：記憶體占用、推論吞吐、精度損失、硬體適配。

4. 什麼是 LoRA/PEFT？和 full fine-tuning 差在哪？  
   - 重點：可訓練參數比例、速度、成本、風險。

5. RAG 的 `chunk size` 和 `overlap` 怎麼選？  
   - 重點：檢索粒度、上下文完整度、重複、索引體積。

6. 你怎麼設計 Prompt？有什麼常見 anti-pattern？  
   - 重點：system/user 區隔、禁止幻覺、輸出 schema、few-shot。

7. LLM 服務的冷啟動與高延遲問題怎麼處理？  
   - 重點：warm-up、batching、限流、快取、降級策略。

8. 怎麼做 LLM 專案的 eval？你會看哪些指標？  
   - 重點：人工評估、任務成功率、重複率、可重現性。

9. 你有沒有遇過資料洩漏（data leakage）？如何發現？  
   - 重點：split 方式、hash 去重、時間切分、泄漏檢查。

10. GPU 只有 4GB 時，如何決定可行模型與 batch 設定？  
   - 重點：模型大小估算、KV cache、上下文長度、併發控制。

## B. ML 基礎與其他模型
1. Embedding/向量檢索為何比關鍵字檢索更好？  
   - 重點：語義相似、向量空間、近似最近鄰、cosine 與內積。

2. 什麼是 cosine similarity？為什麼常用於向量檢索？  
   - 重點：方向比對 vs 幅度差異、正規化。

3. 訓練資料不平衡你會怎麼處理？  
   - 重點：重採樣、class weight、閾值調整、PR-AUC。

4. 分類問題的 Precision / Recall / F1 各在乎什麼？  
   - 重點：誤判成本、商業權衡。

5. 簡述你對 CV 預處理（augmentation / resize / normalization）的認知。  
   - 重點：泛化、過擬合、輸入一致性。

6. 語音模型常見前處理步驟？  
   - 重點：降噪、重採樣、分幀、MFCC。

7. 什麼是過擬合與欠擬合？有何偵測方式？  
   - 重點：訓練/驗證曲線、正則化、早停。

8. 什麼是 ROC-AUC 與 PR-AUC，何時使用各自較合適？  
   - 重點：class imbalance、排序解釋。

9. 你怎麼做 ML 服務漂移（data drift）監控？  
   - 重點：輸入分布、特徵監控、警示門檻。

10. 你會怎麼做推薦/排序任務 baseline？  
   - 重點：先做可解釋 baseline，再逐步加模型複雜度。

## C. 後端 / 系統題（AI 專職常考）
1. 你會怎麼設計一個「多模型推論 API Gateway」？  
   - 重點：路由、佇列、超時、fallback、降載、metrics。

2. API 常見的可用性指標有哪幾個？  
   - 重點：可用率、P95/P99、error rate、吞吐量、資源飽和度。

3. 什麼是佇列（queue）在系統中的角色？  
   - 重點：解耦、削峰、異步處理、死信隊列。

4. 你會怎麼處理併發請求下的共用模型資源？  
   - 重點：模型實例池、排隊、批次、請求降級。

5. 你會如何設計一個最小可用的 Prompt 平台 API？  
   - 重點：認證、版本管理、審計、版本回滾、A/B。

6. 一則「面試官追問」：你剛剛的設計若有一半節點故障怎麼辦？  
   - 重點：重試、超時、冗餘、降級訊息、可觀測性。

## D. 韌體 / C 語言（你關心的架構核心）
1. 你怎麼用 C 設計一個「中斷驅動 + 主迴圈」架構？  
   - 重點：中斷內短小邏輯、主迴圈負責狀態機，避免 blocking。

2. `volatile` 不用在一般變數上的時候會有什麼風險？  
   - 重點：錯誤優化、快取一致性誤判、可讀性與效能平衡。

3. 你怎麼規劃一個韌體的錯誤恢復策略？  
   - 重點：錯誤分類、watchdog、safe mode、可恢復操作。

4. `stack` 和 `heap` 在嵌入式裡為何要慎用？  
   - 重點：碎片、可預測性、可重入、記憶體上限管理。

5. 什麼情況要做 ring buffer？  
   - 重點：串流資料、固定容量、避免動態配置造成碎片。

6. 你會怎麼處理「開機流程」的順序保證？  
   - 重點：clock、外設、驅動、校準、健康檢查。

7. 你怎麼寫韌體時做除錯（斷言、log、trace）？  
   - 重點：可見性、最小侵入、條件化 log、重現性。

8. MCU/RTOS 下的共享資源怎麼保護？  
   - 重點：互斥、關中斷臨界段、優先權反轉與處理。

9. 你怎麼對「功耗、效能、穩定性」做權衡？  
   - 重點：省電模式、週期任務優先級、監控點。

10. 為什麼韌體工程師要知道系統架構，而不只是語法？  
   - 重點：將語法對應到時序與邊界條件，才能預防現場事故。

## E. FPGA 工程（你問的那塊）
1. 你怎麼回答「為什麼這題要上 FPGA 而不是 CPU/GPU」？  
   - 重點：時延、平行度、可重構、功耗/成本權衡。

2. 你怎麼說明 RTL 到位元流（bitstream）的流程？  
   - 重點：RTL、綜合、P&R、timing closure、上板驗證。

3. 時序不收斂（timing fail）常見原因？  
   - 重點：時脈域、comb loop、路徑過長、資源衝突。

4. 你怎麼設計 FPGA 與外部外設的資料通道？  
   - 重點：buffering、握手、時脈同步、回授機制。

5. 為什麼需要 testbench？你怎麼寫最小驗證？  
   - 重點：功能覆蓋、邊界條件、時序場景、回歸測試。

6. 你怎麼讀懂資源報表（LUT/FF/BRAM）？  
   - 重點：瓶頸模組、是否要流水線化、是否可做資源共享。

7. FPGA 上最容易掉坑的「訊號安全」問題是什麼？  
   - 重點：亞穩態、元件同步、reset 可靠性、timing 異常。

8. 你會怎麼做「可升級但可回退」的 bitstream 策略？  
   - 重點：版本管理、雙映像機制、rollback 機制。

9. FPGA 在 AI 場景可以做哪個邊界加速？  
   - 重點：前處理、量化運算、定製化資料管線。

10. 你怎麼解釋 FPGA 的效能指標？  
   - 重點：latency、throughput、Fmax、功耗、資源使用率。
