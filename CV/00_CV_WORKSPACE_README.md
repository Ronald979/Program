# CV Workspace

This folder is now the main resume workspace.

The overall workspace is split into three main areas:

| Folder | Purpose |
|---|---|
| `Learning_Material` | Learning, notes, implementation practice, Q&A, debug records |
| `CV` | Resume descriptions, personal project summaries, project presentation notes |
| `Learning_Roadmap` | Current progress, blockers, next tasks, completed tasks, retrospective |
| `AI_Agent_Monitor` | AI Agent monitoring architecture for start/end task synchronization |

## Files

| File | Purpose |
|---|---|
| `REF_01_Ronald_Tan_CV.pdf` | Current original resume snapshot. Keep it as reference. |
| `01_MASTER_RESUME.md` | Main editable resume source. Update this first. |

## Workflow

1. Keep `REF_01_Ronald_Tan_CV.pdf` unchanged as the original version.
2. Edit `01_MASTER_RESUME.md` whenever a new project or skill stage is completed.
3. Derive targeted resumes from `01_MASTER_RESUME.md` for each job type.

Recommended future files:

| File | Target |
|---|---|
| `resume_fullstack_erp.md` | ERP / full-stack / manufacturing software roles |
| `resume_firmware.md` | firmware / embedded C roles |
| `resume_c_linux.md` | C Linux / embedded Linux roles |
| `resume_fpga_soc.md` | FPGA / SoC software / validation roles |

## Update Principle

Do not add a project only because it was studied. Add it when it has:

- Code or implementation evidence.
- README or architecture explanation.
- Test, validation, benchmark, or debug notes.
- A 2-minute interview story.
